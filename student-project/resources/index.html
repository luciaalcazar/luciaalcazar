Stylesheets & CSS Frameworks

Bootstrap

🌐 No Node.js

Key: Grid, utilities, components.

Use: Quick prototypes, education.

Notes

Very fast to set up; be mindful of visual originality.

Tailwind CSS

📦 Requires Node.js

Key: Utility-first, design tokens, JIT.

Use: Custom design systems.

Notes

Extremely fast once you get it; avoid “class soup”.

Foundation

🌐 No Node.js

Key: Accessibility patterns, robust grid.

Use: Layouts with strong a11y focus.

Notes

Less trendy, but excellent accessibility guidance.

Bulma

🌐 No Node.js

Key: Pure CSS, Flexbox-based.

Use: Lightweight prototypes without JS.

Notes

Clean API; pairs nicely with semantic HTML.

UI Kits & Component Libraries

Material UI

📦 Requires Node.js

Key: React components (Material Design).

Use: Dashboards and portals.

Notes

Very complete; watch bundle size and theming complexity.

shadcn/ui

📦 Requires Node.js

Key: Copy-paste React + Tailwind components.

Use: Portfolios and SaaS landing pages.

Notes

Great consistency with Tailwind; keep tokens centralized.

Ark UI

📦 Requires Node.js

Key: Accessible, headless primitives.

Use: Design systems with full control.

Notes

Excellent a11y defaults; styling is up to you.

Static Site Generators

Jekyll

🟥 Requires Ruby

Key: Markdown + Liquid; friendly with GitHub Pages.

Use: Simple blogs/portfolios.

Notes

Integrates perfectly with GitHub Pages.

Eleventy (11ty)

📦 Requires Node.js

Key: Flexible templates; minimal “magic”.

Use: Lightweight JAMstack sites.

Notes

My ideal balance between control and simplicity.

Hugo

🟢 Standalone binary

Key: Ultra-fast builds; single binary.

Use: Large, content-heavy sites.

Notes

Blazing fast; template syntax has a learning curve.

Astro

📦 Requires Node.js

Key: Islands architecture; multi-framework.

Use: Static sites with selective interactivity.

Notes

Great performance defaults; perfect for case studies.

Critical Reflection

Personal note: This course is very challenging for me — at first I felt like I didn’t understand anything, but little by little I’m starting to get it and make progress.

Workflow fit: I gravitate toward Tailwind + shadcn/ui for speed and visual consistency. For site generation, I pick Eleventy when I want simplicity and full control, or Astro when I need selective interactivity. For high-impact visuals, GSAP is my go-to.

Trade-offs: Speed vs. Flexibility — templates help me move fast, but can limit originality, so I customize them. Dependencies vs. Simplicity — Node toolchains are powerful, but add setup and complexity; sometimes a simple CDN drop-in is better. Node.js vs. Standalone — for quick classroom deliverables, standalone/CDN tools are ideal; for long-term projects, Node can pay off. Creativity vs. Convention — I reuse proven patterns without losing my own visual voice.

Portfolio use: I would use Eleventy or Hugo for editorial and content-heavy projects (e.g., Specimen), and Astro for interactive case studies. I’ll keep Tailwind tokens consistent across projects to maintain a cohesive identity.
