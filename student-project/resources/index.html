Stylesheets & CSS Frameworks

Bootstrap

🌐 No Node.js

Key: Grid, utilities, components.

Use: Prototipos rápidos, educación.

Notes

Muy rápido de montar; hay que cuidar la personalidad visual.

Tailwind CSS

📦 Requires Node.js

Key: Utility-first, tokens, JIT.

Use: Sistemas de diseño a medida.

Notes

Velocidad brutal una vez interiorizado; evitar “class soup”.

Foundation

🌐 No Node.js

Key: Accesibilidad, grid robusto.

Use: Layouts con foco en a11y.

Notes

Menos popular, pero criterios de a11y muy sólidos.

Bulma

🌐 No Node.js

Key: Solo CSS, basado en Flexbox.

Use: Prototipos ligeros sin JS.

Notes

API limpia y fácil; acompaña bien a HTML semántico.

UI Kits & Component Libraries

Material UI

📦 Requires Node.js

Key: Componentes React (Material Design).

Use: Dashboards y portales.

Notes

Muy completo; ojo con el tamaño de bundle y theming.

shadcn/ui

📦 Requires Node.js

Key: Componentes React + Tailwind copiables.

Use: Portfolios y landings SaaS.

Notes

Gran consistencia con Tailwind; centralizar tokens.

Ark UI

📦 Requires Node.js

Key: Primitivas accesibles, headless.

Use: Sistemas de diseño con control total.

Notes

Excelentes defaults de accesibilidad; el estilo va por tu cuenta.

Static Site Generators

Jekyll

🟥 Requires Ruby

Key: Markdown + Liquid, GitHub Pages friendly.

Use: Blogs/portfolios simples.

Notes

Se integra perfecto con GitHub Pages.

Eleventy (11ty)

📦 Requires Node.js

Key: Plantillas flexibles; poca “magia”.

Use: JAMstack ligero.

Notes

Mi equilibrio ideal entre control y simplicidad.

Hugo

🟢 Standalone binary

Key: Builds ultrarrápidos; binario único.

Use: Sitios grandes o con mucha info.

Notes

Velocísimo; la sintaxis de plantillas requiere curva inicial.

Astro

📦 Requires Node.js

Key: Arquitectura de islas; multi-framework.

Use: Sitios estáticos con interactividad selectiva.

Notes

Grandes defaults de rendimiento; ideal para estudios de caso.

Animation & Visual Libraries

GSAP

🌐 No Node.js

Key: Timelines, ScrollTrigger.

Use: Storytelling interactivo.

Notes

Calidad profesional, docs excelentes.

Framer Motion

📦 Requires Node.js

Key: Animación para React.

Use: UI polish en portfolios/SaaS.

Notes

API expresiva; encaja con el modelo mental de React.

Anime.js

🌐 No Node.js

Key: Motor ligero de animación.

Use: Micro-interacciones e ilustraciones.

Notes

Ideal para efectos rápidos sin setup pesado.

AOS

🌐 No Node.js

Key: Animaciones on-scroll.

Use: Landings y presentaciones.

Notes

“Drop-in delight”; usar con moderación por accesibilidad.

Templates & Kits

HTML5 UP

🌐 No Node.js

Key: Plantillas responsive gratuitas.

Use: Despliegues de estudiante y rápidos.

Notes

Buen punto de partida; conviene mejorar semántica.

Flowbite

📦 Requires Node.js

Key: Librería de componentes para Tailwind.

Use: Montaje rápido de UI con Tailwind.

Notes

Ahorra tiempo; revisar a11y en combos personalizados.

Cruip

📦 Requires Node.js (React)

Key: Templates premium/free (React/Tailwind).

Use: Landing de startups.

Notes

Estética cuidada; revisar licencias para clase.

Data & Media Services

Hygraph

☁ SaaS

Key: Headless CMS con GraphQL.

Use: Gestión de contenido estructurado.

Notes

Ideal cuando el modelo de contenido importa; aprender lo básico de GraphQL.

ImageKit.io

☁ SaaS

Key: CDN de imágenes, optimización y transforms.

Use: Rendimiento, responsive media.

Notes

Enchufar y listo; subidón de Lighthouse.

Netlify CMS

🌐 No Node.js

Key: CMS basado en Git con interfaz web.

Use: CMS para sitios estáticos.

Notes

Experiencia de editor sin servidores; perfecto para equipos pequeños.

Critical Reflection (ATELIER)

Workflow fit: Tailwind + shadcn/ui para velocidad y consistencia; Eleventy o Astro según la interactividad; GSAP para secciones de alto impacto.

Trade-offs: Speed vs. Flexibility — las plantillas aceleran, pero pueden limitar originalidad. Dependencies vs. Simplicity — Node aporta poder, pero también setup. Node.js vs. Standalone — para entregas rápidas en clase, mejor standalone/CDN. Creativity vs. Convention — usar librerías sin perder voz propia.

Portfolio use: Eleventy/Hugo para piezas editoriales (como Specimen); Astro para casos interactivos; mantener tokens de Tailwind para coherencia visual entre proyectos.
